## 项目介绍

VPS Box 是一个面向 Linux VPS 场景打造的综合型运维脚本。  
它希望解决的不是某一个单点问题，而是把一台服务器从**初始化、调优、安全加固，到节点部署与后续维护**这一整套流程，尽可能统一到一个交互式工具里完成。

和很多只负责“装一个协议”或者“改几条内核参数”的脚本不同，VPS Box 更像是一套适合长期使用的 VPS 管理入口：

- 新机拿到手后，可以先做系统维护和基础安全配置
- 网络表现不理想时，可以直接做 TCP / BBR 优化
- 需要代理服务时，可以部署常见主流节点协议
- 需要长期维护时，还能继续管理防火墙、SSH、定时任务、磁盘和常用工具

它的目标不是做一堆零散命令的简单拼接，而是尽量把常见 VPS 维护场景整理成**清晰、直接、可重复使用的菜单式操作体验**。

---

## 这个项目解决什么问题

很多人在使用 VPS 时，都会遇到同一类问题：

- 系统初始化步骤杂，装完系统还要手动做很多基础配置
- 节点部署脚本很多，但只管安装，不管后续维护
- 加一个节点、删一个节点，经常要自己改配置文件
- 证书、端口、防火墙、SSH、内核优化这些操作分散在不同工具里
- 小白很难记住一堆 Linux 命令，老手也不想每次重复做机械配置

VPS Box 把这些高频动作集中到一个脚本中，提供一个统一的 VPS 管理入口。  
简单说，它想做的是：

**一套脚本，尽可能覆盖一台 VPS 在真实使用中的大部分常见运维需求。**

## 核心特点

### 1. 不只是节点脚本，而是完整 VPS 管理工具

VPS Box 并不是单纯的“翻墙节点安装器”。

它把以下能力整合在了一起：

- 系统更新、清理、时区、主机名、Swap、DNS
- TCP 调优、BBR 管理、流媒体 / IP 质量检测
- Docker、Fail2Ban、WARP、UFW 防火墙
- SSH 密钥登录管理
- 磁盘挂载 / 格式化 / 检查
- 定时任务管理
- 常用工具箱安装
- 多协议节点部署与维护
- PicVault 图床部署与维护

所以它更适合作为一台 VPS 的“总控台”，而不是一次性安装器。

### 2. 面向真实使用场景设计

这个项目不是为了展示参数有多花哨，而是偏向实际好用：

- 小白可以通过菜单直接完成大多数操作
- 老手可以省去重复性的初始化步骤
- 一台服务器的基础维护和节点管理能放在一起做
- 后续维护时不需要再到处找不同脚本

### 3. 节点管理更注重可维护性

VPS Box 当前的节点部分，不再只是“生成一份配置然后结束”。

它更强调：

- 节点可继续查看
- 节点可删除
- HY2 端口跳跃规则随节点自动创建 / 清理
- 配置可备份
- 配置可恢复
- 节点信息可长期管理

这让它比很多“一次部署完就不管了”的脚本更适合长期使用。

### 4. 尽量降低误操作成本

在很多高风险环节里，VPS Box 的设计偏保守：

- 防火墙操作会优先考虑 SSH 端口
- 节点增删时尽量避免粗暴重启
- 证书申请失败时会处理残留
- 80 端口被占用时不会强行乱杀服务

这意味着它不是那种“看起来很猛但容易翻车”的脚本，而是更偏向**稳一点、少出事**的路线。

## 项目适合谁

VPS Box 适合以下几类用户：

### 新手用户

如果你对 Linux 命令不熟，但又希望能自己管理 VPS，这个项目会比较友好：

- 菜单式操作更容易理解
- 省去大量手敲命令
- 节点部署、系统维护、安全管理都集中在一起

### 进阶用户

如果你已经会基础 VPS 操作，但不想每次重复做这些杂务，它也很合适：

- 新机初始化更快
- 常见运维能力不用再拆成多个脚本
- 节点和系统管理能放在同一入口里完成

### 长期维护多功能 VPS 的用户

如果你的服务器既要跑节点，又要做基础运维，这个项目会更顺手：

- 能同时处理系统层和代理层需求
- 更适合长期维护，而不是一次性安装后丢着不管

## 节点能力概览

在节点能力上，VPS Box 支持当前主流的几类常见方案：

- **VLESS-Reality**
- **VLESS-WS-TLS**
- **AnyTLS**
- **Hysteria2**
- **Shadowsocks**
- **快捷中转 / 端口转发管理**

其中 Hysteria2 支持可选 UDP 端口跳跃：开启后会自动写入 `mport=起始端口-结束端口` 分享参数，并创建对应的 UDP REDIRECT 规则；删除节点时也会自动清理该节点对应的跳跃规则，避免残留防火墙配置。

快捷中转 / 端口转发支持添加、查看和删除，入口在「节点管理 → 端口转发管理」。删除转发时会同步清理 VPSBox 创建的 iptables DNAT / MASQUERADE 规则或 socat systemd 服务。

证书管理入口在「节点管理 → 证书管理」，支持查看 VPSBox 安装目录 `/etc/vpsbox-cert` 与 acme.sh 默认目录 `/root/.acme.sh` 中的证书、检查过期时间与 SAN、手动续签、删除证书和 acme.sh 记录。

并支持围绕 Xray / Sing-box 进行部署和管理。

它的重点不只是“能装”，还包括：

- 查看已部署节点
- 输出分享链接
- 生成二维码
- 删除指定节点
- 备份节点配置
- 从历史快照恢复节点配置

这让节点功能从“一键安装”扩展成了“一键管理”。

## 系统与运维能力概览

除了节点之外，VPS Box 还覆盖了大量日常运维能力，例如：

- 系统更新与垃圾清理
- DNS 优化与时区设置
- SSH 端口与 SSH 密钥管理
- Swap 管理
- TCP 网络优化与 BBR 管理
  - 内置动态 TCP 画像生成：按本地带宽 / 服务器带宽 / RTT / 内存自动选参
  - 高延迟画像带缓冲上限与窗口缩放抑制，避免中高 RTT 过早顶满 `tcp_adv_win_scale` 或 `rmem_max`
  - 生成后会写入 sysctl / limits / systemd 配置，并可直接查看当前画像
- Docker 安装
- PicVault 图床部署、更新、状态查看、日志、重启、停止、备份和卸载
- Fail2Ban 防暴力破解
- Cloudflare WARP
- UFW 防火墙管理
- 磁盘分区挂载与格式化
- 定时任务管理
- 常用运维工具箱安装

换句话说，它不是只管“代理”，而是把 VPS 上常见的基础操作一起打包了。

## 为什么这个项目有价值

VPS 相关脚本很多，但常见问题也很明显：

- 功能单一，只解决一个点
- 部署之后缺少后续管理能力
- 配置分散，维护成本高
- 对新手不够友好
- 对老手又不够省事

VPS Box 的价值在于，它试图在“简单上手”和“长期可用”之间找到平衡：

- 对新手来说，它降低了 VPS 使用门槛
- 对老手来说，它减少了重复劳动
- 对长期维护来说，它让系统与节点管理集中在一起

所以这个项目更像是一个 **VPS 综合运维工作台**。

## 使用方式

### 在线运行

```bash
bash <(curl -sL https://raw.githubusercontent.com/vmenzo/VPSBox/main/vpsbox.sh)
```

兼容传统写法：

```bash
curl -fsSL https://raw.githubusercontent.com/vmenzo/VPSBox/main/vpsbox.sh | bash
```

首次运行后会自动注册快捷命令：

```bash
vpsbox
```
